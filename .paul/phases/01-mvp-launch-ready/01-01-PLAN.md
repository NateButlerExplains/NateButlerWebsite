<objective>
## Goal

Build the commanding hero section as the visual system anchor for the entire Cyber Command Center brand. The hero establishes the motion language, color application hierarchy, asymmetry patterns, spacing standards, and neon glow effects that all subsequent components will inherit. This is not just a page section—it's the design system baseline made visible.

## Purpose

The hero must be perfect before navigation and about components are built. It defines:
- **Motion language:** How Framer Motion animations feel and behave across the site
- **Color hierarchy:** Primary (cyan), secondary (purple), tertiary (magenta) in context, where each is used
- **Asymmetry blueprint:** 60/40 splits, off-center placement, overlapping boundaries, negative space usage
- **Glassmorphism baseline:** Opacity, blur values, how glass interacts with content and depth
- **Neon language:** When to apply glows, glow intensity, layering, shadow vs. glow hierarchy
- **Spacing & typography:** How headlines, copy, and CTAs relate spatially and in the layout grid
- **Performance ceiling:** LCP <2.5s constraint that all animations must respect without compromise

Once the hero is locked and approved, Navigation (PLAN 01-02) and About (PLAN 01-03) will use hero as design reference—copying its motion patterns, color applications, spacing rhythm, and visual effects.

## Output

- `src/components/hero/hero-section.tsx` — Hero component with full Framer Motion animation, asymmetric layout, dual CTAs
- `src/components/hero/hero-animation.ts` — Extracted animation constants, keyframe definitions, timing values (reusable for other components)
- `src/components/ui/button.tsx` — Button component (primary, secondary, ghost variants) as they appear in hero context
- `src/app/page.tsx` — Home page rendering hero as the complete above-the-fold experience
- Design system reference document (comments in hero-animation.ts) capturing motion language, color patterns, spacing rhythm
</objective>
